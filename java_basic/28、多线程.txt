
要想了解多线程必须先了解线程，再先了解进程线程依赖于进程而存在
进程：
	通过任务管理器看到进程的存在,通过观察发现只有运行的程序才会出现进程
	进程：正在执行的程序，
	是系统进行资源分配和调用的独立单位，每一个进程都有它自己的内存空间和系统资源
多进程有什么意义？
	单进程的计算机只能做一件事，多进程的计算机在一个时间段内执行多个进程。提高CPU 的使用率。

什么是线程？
	字一个进程内可以执行多个任务，每个任务是一个西纳城
	线程：程序的执行的单元，程序的执行路径，是程序使用CPU的最小单位
	一条程序有多条执行路径，就是多线程
多线程的意义？
	多线程的存在不是为了提高程序的执行速度，是为了提高程序的使用率

	并行：逻辑上同时发生，指某一时间内同时运行多程序。
	并发：物理上同时发生，指某一个时间点内同时运行多程序。

java不能直接调用系统功能，java可以直接调用C/C++写好的程序来实现多线程
	第一种是将MyThread类声明为Thread的子类,
		该子类重写run()方法,被线程执行的代码比较耗时	
		创建对象
		启动线程 start()
	run 方法和start方法的区别：
	run()：仅仅是封装了被线程执行的代码，直接调用是普通方法
	start()：首先启动线程，然后由jvm调用该线程的run方法
	illegalThreadStateException

	获取线程的名称：
	public final String getName()
	设置线程名称：
	public static void setName(String name)
	public static THread currentThread()
	Thread.currentThread().getName(); //获取main线程
